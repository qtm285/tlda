The Union Bound

Placeholder

The union bound states that for any events \(A\) and \(B\), \[ \P(A \cup B) \le \P(A) + \P(B). \]

More generally, for events \(A_1 \ldots A_k\), \[ \P(A_1 \cup \cdots \cup A_k) \le \P(A_1) + \cdots + \P(A_k). \]

This follows from inclusion-exclusion: \(\P(A \cup B) = \P(A) + \P(B) - \P(A \cap B)\), and the intersection has non-negative probability.

Markov’s Inequality and Consistency

Convergence in mean square implies convergence in probability. Let’s use Markov’s inequality to see why.

Markov’s Inequality

Markov’s inequality says that for a non-negative random variable \(X\) and any \(t > 0\), \[ P(X \ge t) \le \frac{\mathop{\mathrm{E}}[X]}{t}. \]

The usual proof of Markov’s inequality is based on a few simple observations.

  1. The expectation of the indicator variable \(1_{\ge t}(X)\) is the probability that \(X\) exceeds \(t\), \(P(X \ge t)\).
  2. If we have some function of \(u\) that’s always larger than \(1_{\ge t}\), i.e., one satisfying \(u_t(x) \ge 1_{\ge t}(x)\) for all \(x\), we know that \(\mathop{\mathrm{E}}[u_t(X)] \ge \mathop{\mathrm{E}}1_{\ge t}(X)\) for any random variable \(X\). If it’s always larger for non-negative \(x\), then \(\mathop{\mathrm{E}}[u_t(X)] \ge \mathop{\mathrm{E}}[1_{\ge t}(X)]\) for any non-negative random variable \(X\).
  3. The function \(u_t(x)=x/t\) is such a function.1
Proving Markov’s Inequality

Prove Markov’s inequality.

Solution

To see that \(u_t(x) = x/t \ge 1_{\ge t}(x)\) for all \(x \ge 0\), consider separately the cases \(x \ge t\) and \(x < t\). If \(x \ge t\), then \(1_{\ge t}(x)=1\) and \(x/t \ge 1\). If \(x < t\), then \(1_{\ge t}(x)=0\) and \(x/t \ge 0\). So \(u_t(x) \ge 1_{\ge t}(x)\) for all non-negative \(x\). It follows that for any non-negative random variable \(X\), \[ P(X \ge t) = \mathop{\mathrm{E}}[1_{\ge t}(X)] \le \mathop{\mathrm{E}}[u_t(X)] = \mathop{\mathrm{E}}\qty[\frac{X}{t}] = \frac{\mathop{\mathrm{E}}[X]}{t}. \]

Often, instead of using this to bound the random variable we’re interested in directly, e.g. \(X=|\hat\theta - \theta|\), we use it to bound the random variable’s square. \(|X| \ge \epsilon\) if and only if \(X^2 \ge \epsilon^2\), so the probability that \(|X| \ge \epsilon\) is the same as the probability that \(X^2 \ge \epsilon^2\). Applying Markov’s inequality to the random variable \(X^2\) gives us a bound in terms of \(X\)’s mean square which, in the specific case that \(X\) is \(|\hat\theta-\theta|\), is the mean squared error of the estimator \(\hat\theta\), \(\RMSE(\hat\theta)^2=\mathop{\mathrm{E}}[(\hat\theta-\theta)^2]\)

\[ P(X \ge \epsilon) = P(X^2 \ge \epsilon^2) \leq \frac{\mathop{\mathrm{E}}[X^2]}{\epsilon^2} \qqtext{ e.g.} P(|\hat\theta - \theta| \ge \epsilon) = P((\hat\theta - \theta)^2 \ge \epsilon^2) \leq \frac{\mathop{\mathrm{E}}[(\hat\theta - \theta)^2]}{\epsilon^2} \]

This tells us that, if the root-mean-squared error of \(\hat\theta\) goes to zero, then the probability that \(\hat\theta\) is any distance \(\epsilon\) away from \(\theta\) goes to zero, i.e., consistency in mean-square implies consistency in probability.

Markov’s Inequality and Interval Estimation

So far, when we’ve calibrated interval estimates using our estimator’s standard deviation, we’ve relied on normal approximation. In effect, we’ve been using a formula for \(P(\lvert\hat\theta - \theta\rvert \le \epsilon)\) that’s accurate when \(\hat\theta\) has a normal distribution and close enough when its distribution is close enough to normal. In this problem, we’re going to think about doing without this reliance on approximate normality.

Let’s consider \(\hat\theta\), an unbiased estimator of \(\theta\) with standard deviation \(\sigma\), so the normal approximation to the distribution of \(\hat\theta-\theta\) has the density \(f_{0,\sigma}(x)\) below.

\[ P\qty(|\hat\theta - \theta| \le \epsilon) \approx \int_{-\epsilon}^{\epsilon} f_{0,\sigma}(x) dx \qfor f_{0, \sigma}(x) = \frac{1}{\sqrt{2\pi}\sigma} e^{-x^2/2\sigma^2} \]

The reason we’ve been talking about interval estimators of the form \(\hat\theta \pm 1.96 \sigma\) is that, if this approximation were perfect, these interval estimators would have 95% coverage. That is, it’d be true that \(P(|\hat\theta-\theta| \le 1.96 \sigma) = .95\). And if the approximation is pretty good, we should still expect coverage close to that. But suppose we’re not confident that it is. Markov’s inequality allows us to calibrate interval estimates in terms of our estimator’s standard deviation without any caveats about its sampling distribution being approximately normal. Let’s give it a shot.

Exercise 20.1  

Let \(\hat\theta\) be an unbiased estimator of \(\theta\) with standard deviation \(\sigma\). By applying Markov’s inequality to \(|\hat\theta-\theta|^2\), find a lower bound on \(P(|\hat\theta-\theta| \le t\sigma)\) as a function of \(t\). For what choice of \(t\) is this bound equal to \(.95\)? Explain how you can use this to get an interval estimate \(\hat\theta \pm t\sigma\) with a coverage probability of at least 95%.

From Upper Bounds to Lower Bounds.

If you have an upper bound on \(P(X > \epsilon)\), then you have a lower bound on \(P(X \le \epsilon)\). \(P(X \le \epsilon)=1-P(X > \epsilon)\) because \(X > \epsilon\) if and only if it is not the case that \(X \le \epsilon\), so if \(P(X > \epsilon) \le u\), then \(P(X \le \epsilon) = 1-P(X > \epsilon) \ge 1-u\). And if you’re troubled that Markov’s inequality says \(\ge\) instead of \(>\), don’t be. You have a bound on \(P(X > \epsilon)\) too. \(P(X > \epsilon) \le P(X\ge \epsilon)\) because \(X \ge \epsilon\) whenever \(X > \epsilon\).

Solution

Applying Markov’s inequality to \(|\hat\theta-\theta|^2\), which has mean-square \(\sigma^2\) because \(\hat\theta\) is unbiased, for\(\epsilon = (t\sigma)^2\) gives

\[ P(|\hat\theta-\theta|^2 \ge (t\sigma)^2) \le \frac{\mathop{\mathrm{E}}[(\hat\theta-\theta)^2]}{(t\sigma)^2} = \frac{\sigma^2}{(t\sigma)^2} = \frac{1}{t^2}. \] The probability that \(|\hat\theta-\theta| > t\sigma\) is also no larger than \(1/t^2\) because this happens only if \(|\hat\theta-\theta|^2 \ge (t\sigma)^2\). And it follows that the probability that \(|\hat\theta-\theta| \le t\sigma\) is greater than \(1-1/t^2\).

To find the value of \(t\) that makes this lower bound equal to \(.95\), we solve \(1-1/t^2=.95\) to find \(t=\sqrt{1/(1-.95)}= \sqrt{20} \approx 4.5\). And because \(\hat\theta \in [\theta - t\sigma, \theta + t\sigma]\) if (and only if) \(|\hat\theta-\theta| \le t\sigma\), it follows that this interval has a coverage probability of at least \(.95\).

I’ve made a separate plot for each of the two point estimators, showing the sampling distribution as a gray bar graph, the bootstrap sampling distribution as an orange bar graph, the normal approximation to the sampling distribution as a black line, and three interval estimates. library(tidyverse)

prior.obs.estimator = function(Y) { (theta.prior*n.prior + sum(Y)) / (n.prior + n) } 

sampling.distribution = function(estimator) {
    draws = 1:10000 |> map_vec(function(.) {
        J = sample(1:m, size=n, replace = TRUE)
        Y = y[J]
        estimator(Y)
    })
}

bootstrap.sampling.distribution = function(estimator, draws=10000) {
    1:draws |> map_vec(function(.) {
        I = sample(1:n, size=n, replace = TRUE)
        Y.star = Y[I] 
        estimator(Y.star)
    })
}

To calculate coverage, I ran this code. Because we have to bootstrap our estimator for each sample drawn from our population, computation time can get out of control if we use 10k samples from the population and 10k draws from the bootstrap sampling distribution as usual. I used 1000 of each, which is about 100 times faster.

library(tidyverse)

bootstrap.width = function(estimator) {
    bootstrap.draws = bootstrap.sampling.distribution(estimator, draws=1000)
    width(bootstrap.draws)
}
normal.width = function(estimator) {
    theta.hat = estimator(Y)
    if(identical(estimator, prior.obs.estimator)) {
        2*1.96*sqrt(n*theta.hat * (1-theta.hat) / (n.prior+n)^2)
    } else {
        2*1.96*sqrt(theta.hat * (1-theta.hat) / n)
    }
}

coverage = function(estimator) {
    1:1000 |> map(function(.) {
        J = sample(1:m, size=n, replace = TRUE)
        Y = y[J]
        point.estimate = estimator(Y)

        bootstrap.covers = abs(point.estimate - theta) <= bootstrap.width(estimator)/2 
        normal.covers = abs(point.estimate - theta) <= normal.width(estimator)/2  
        markov.covers = abs(point.estimate - theta) <= (sqrt(20)/1.96)*normal.width(estimator)/2 
        data.frame(bootstrap=bootstrap.covers, normal=normal.covers, markov=markov.covers)
    }) |> bind_rows() |> 
          summarize(across(everything(), mean))
}

mean.coverage = coverage(mean)
prior.obs.coverage = coverage(prior.obs.estimator)

The clear takeway, from an interval estimation perspective, is that calibration via bootstrap or normal approximation is a better idea than calibration using Markov’s inequality. As established in Exercise exr-markov-interval, intervals calibrated using Markov’s inequality are roughly \(\sqrt{20}/1.96 \approx 2.3\) times wider. The question of which point estimator to use is a little more subtle. Here the prior-observations estimator looks pretty good. We get substantially narrower intervals and the coverage probability of our bootstrap and normal-approximation-calibrated intervals is close to the nominal level—it’s 90%. But we know from our discussion in the last homework and the bias/coverage lecture that using this estimator can be risky. We’ve managed to do well because our prior mean \(\thetaprior \approx 0.52\) was close to the population mean \(\theta \approx 0.49\) and we haven’t used it ‘too much’. In fact, we’re in a pretty similar situation to the one I plotted in the last row in my solution to Exercise exr-bias-variance-rmse — we have \(n=100\), \(\theta\approx .5\), and \(\theta-\thetaprior\approx .025\) — where using \(\nprior=100\) as we have here is a substantial improvement relative to \(\nprior=0\) in terms of root-mean-squared error and not-too-unreasonable inferentially because it gave us a \(\text{bias}/\text{sd}\) ratio of roughly \(1/2\). That said, I wouldn’t have been comfortable using it because I wouldn’t have known I was in this situation without knowing the actual value of the estimation target \(\theta\). If it had been the case that \(\theta=.58\), which is consistent with what our 95% interval for the sample mean tells us is possible, then we’d have been in a similar situation to the one I plotted in the first row in my solution to Exercise exr-bias-variance-rmse — we’d have had a \(\text{bias}/\text{sd}\) ratio of roughly \(2\) — and the result would be 50% coverage.

Note that here what I’m doing when I talk about these plots is a doing-the-math-by-hand version of testing out an estimator on data I’ve simulated by drawing samples from a few different fake populations. What I’m saying is, in essence, that the estimator \(\tilde Y_{\nprior}\) works well in some populations and not in others, and since I don’t know which will happen in my actual population, I don’t feel comfortable using it.

I’d have been more comfortable using it at the larger sample size \(n=400\), since having more actual observations waters down the influence of the \(100\) prior observations. I still wouldn’t have chosen it myself, as once the sample size gets large enough that the potential for bias starts to erode, the interval-narrowness advantage erodes too. Here’s a version of the plot I made in Exercise exr-bias-variance-rmse for \(n=400\) with \(\thetaprior=.5\) and \(\theta=.6\), where you can see that at \(\nprior=100\) we have a bias/sd ratio of \(1\), which is ok but not great, but also a pretty minimal reduction in standard deviation relative to using the sample mean (i.e. \(\nprior=0\)).

The difference this larger sample size would make for me is that I’d be less inclined to disagree with someone else’s choice to use \(\tilde Y_{\nprior}\), as I’d be content knowing that there isn’t much difference between it and the sample mean. Here the context does matter. I can be pretty confident that \(\theta\) is close to \(.5\) because, while it’s not exactly the average win/game ratio of teams in the NBA (which has to be \(.5\) because one team wins and one loses in each game), it should be pretty close. It’s not going to further out than \(.4\) or \(.6\). If it were plausible that \(\thetaprior\) differed from \(\theta\) more, we’d have to water down our prior observations with more real ones to ensure the prior observations weren’t doing much.
